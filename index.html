单步调试

  • step over next function call
  • 跳过接下来一整个函数执行
  • 你认为接下来的这个函数是没问题的
  • step into next function call
  • 进入接下来的一个函数体
  • 你认为接下来的这个函数可能存在问题
  • step out of current function
  • 退出当前函数体
  • 你进去看了看，估计没啥问题

代理调试

什么是代理？

  • 你想认识 TA，TA 收到各种小纸条并积极回应，这叫“web 服务”
  • 你想传纸条给 TA，需要别人帮你传纸条，这叫 “路由”
  • TA 不知道你让你的朋友 A 审查你给 TA 的纸条，这叫“正向代理”
  • 你不知道 TA 让 TA 的朋友 B 审查 TA 接收传出的纸条，这叫“反向代理”
  • 你的纸条必须经过 C，但你和 TA 都不知道 C 在偷偷审查纸条，这叫“透明代理”
  • 凡是你传来的纸条，B 都要打开看看有多肉麻，你这是被匹配了“代理规则”
  • A, B, C 能做什么？细思恐极

let mergeCORSHeader = (reqHeader, originHeader) => {
	var targetObj = originHeader || {};

	delete targetObj['Access-Control-Allow-Credentials'];
	delete targetObj['Access-Control-Allow-Origin'];
	delete targetObj['Access-Control-Allow-Methods'];
	delete targetObj['Access-Control-Allow-Headers'];

	targetObj['access-control-allow-credentials'] = 'true';
	targetObj['access-control-allow-origin'] = reqHeader['origin'] || '*';
	targetObj['access-control-allow-methods'] = 'GET, POST, PUT';
	targetObj['access-control-allow-headers'] = reqHeader['access-control-request-headers'] || '*';

	return targetObj;
};

反向代理配置


const $browserSync = require('browser-sync');
const $proxy = require('http-proxy-middleware');

$gulp.task('serve-browserSync', done => {
	// 默认我们包裹 mock 服务来进行调试
	// 也可以包裹线上服务地址来进行调试
	// borwserSync 结合 http-proxy-middleware 能获得更强大的反向代理功能
	$browserSync({
		ghostMode: false,
		injectChanges: true,
		port: $config.devServerPort,
		proxy: {
			target: '127.0.0.1:' + $config.mockServerPort
		},
		middleware: [
			{
				route: '/XianFront',
				handle: $proxy({
					target: 'http://test.dsx.qq.com/',
					changeOrigin: true,
					logLevel: 'debug',
					secure: false
				})
			},
			// 反向代理示例
			// 到 http://127.0.0.1:8090/index.php 的请求，都会转发到
			// http://m.db.house.qq.com/index.php
			{
				route: '/index.php',
				handle: $proxy({
					target: 'http://openapi.house.qq.com',
					changeOrigin: true,
					logLevel: 'debug',
					secure: false
				})
			}
		]
	});
	done();
});

调试环境总结

  • 目标是可以模拟任意状况，任意步骤
  • hosts，代理，反向代理，mock 服务 需要灵活搭配
  • 前后分离可减少沟通成本

如何节省调试开销?

使用 Linter

  • http://eslint.cn/
  • 自定义适合自己的约束规则
  • eslint-plugin-angular
  • sublime linter

积累 HACK

  • 例子: animation 动画效果异常
  • backface-visibility: hidden;

遵守规范

  • 例子: 网页编码应当使用utf8
  • ㎡ 与 m²
  • ㎡: \u33a1 gbk支持
  • ²: \u00B2 gbk不支持，无法显示

完善工具

  • 长期坚持，不断更新
  • 例子: 混用换行符引发的页面输出故障
  • unix换行符 \CR
  • windows换行符 \CR\LF

培养好习惯

  • 让代码趋于更加合理，而不是能工作就行
  • 例子: 不要滥用 position:relative
  • 真实案例: 初始化白屏，见于 android uc 浏览器
